Learning HTML

Learning some HTML!

Creating Your First Webpage

Now, to get a good hand at HTML, what better way is there than to make your own website using HTML coding! Use the editor on the right hand side to get started.

Start by entering the <html> tag at the beginning of the document. This essentially tells the browser that the this file is an HTML file. Although, in some respects this tag is becoming obsolete, it is always good practice to include it. On the next line add two spaces and the <body> tag. The two spaces are to have 'whitespacing'; an element of HTML that makes code easier to read and understand, whereas the the body tag is to tell the browser that the upcoming code is to be displayed in the 'body' of the page. Line 3: Add 4 spaces (for whitespacing) and then the code <h1>HELLO WORLD!</h1>. This code tells the browser to show the text "HELLO WORLD!" in the form of a primary heading, hence the name H1 or Heading 1. Now, you are probably wondering what the second <h1> tag is with the forward slash in front is. This tag, </h1>, signifies the end of an 'element'; like the fullstop at the end of a sentence. Now, to wrap-up your webpage all you need to do is, on the next line, add 2 spaces and </body>, and on the final line: </html>.

And, voila! Take a look at your very first HTML webpage! Once your done, you can have a fiddle around with the code, to modify it!

Common HTML Tags

We have already learnt about some key important HTML tags, however, here are some tags that will prove to be useful in your HTML coding endeavours.
But first, lets step through the anatomy of an element!

<head>


This tag, the 'head' tag, represents everything that the reader cannot see, for example links to fonts and CSS files, also known as stylesheets, titles, and the favicon (the icon you see on the tab of a website). It can also contain the <meta> tag; a tag which delivers metadata. In simple terms, the <meta> tag can tell the browser and user information about the website; keywords, a description, author, or many other attributes of the website.

<html>
  <head>
    <title>EXAMPLE</title>
  </head>
<html>

<body>


The body, essentially, is a tag that encompasses everything within the website that the viewer can read or see. This can include paragraphs, media, tables, lists, and lots of other web elements. These can be explored online, as there are simply too many to list!

<html>
  <head>...</head>
  <body>
    <p>...</p>
  </body>
<html>

<h1>, <h2>, <h3>, <h4>, <h5>, and <h6>


These tags represent, respectively, primary, secondary, tertiary, quaternary, quinary, and/or senary headings. Depending on what you need, the purposes of each will become obvious. The tier of heading also determines the font weight (boldness), size, and thus, how much it will stand out.

<html>
  <head>...</head>
  <body>
    <h1>I am a Heading 1</h1>
    <h2>I am a Heading 2</h2>
    <h3>I am a Heading 3</h3>
    <h4>I am a Heading 4</h4>
    <h5>I am a Heading 5</h5>
    <h6>I am a Heading 6</h6>
  </body>
<html>

<p>


This tag will typically form the basis of your website. The <p> tag is for the creation of paragraphs, hence its name the 'p' tag.

<html>
  <head>...</head>
  <body>
    <p>I am a paragraph</p>
  </body>
<html>

<em>, <u>, and <strong>


These are styling tags that are inserted within a text tag, e.g. <h1> or <p>. <em> is used to put emphasis on a tag through italics and slight bolding. <u> puts and underline under the text it encloses, and <strong> makes the text bold.

<html>
  <head>...</head>
  <body>
    <em>I am emphasised</em>
    <u>I am underlined</u>
    <strong>I am bold or strong</bold>
  </body>
<html>

<ol> and <ul>


Lists are a part of website creation that is extremely important. Not only are they for normal lists, they can also be used for creating navbars and many other web elements. You can have 2 types of lists. <ul> stands for unordered list. In an unordered list the contents are displayed as bullet points. <ol> stands for ordered list. In an ordered list the contents are automatically ordered. In any list (<ol> or <ul>) you have to use the <li> tag. Each <li> tag represents a new bullet point.

<html>
  <head>...</head>
  <body>
    <ol>
      <li>I am an ordered list item</li>
    </ol>
    <ul>
      <li>I am an unordered list item</li>
    </ul>
  </body>
<html>

<img>


Images are a fundamental aspect of making an attractive webpage. Images are able to represent what you would like to say, and as the saying goes: “a picture can tell a thousand words”. To insert an image in your webpage, you must use the <img> tag, and as it is an inline element, it must be enclosed within another tag. Within this tag you will have a src attribute and an alt attribute. The src attribute is what tells the server where to find the image. The alt attribute tells the server what to replace the image with if the users browser cannot load up the image. The alt attribute also provides the browser with a desciption of the picture for accessibility reasons.
<html>
  <head>...</head>
  <body>
    <img src="YOUR/IMAGE.jpg" alt="This is your image description...">
  </body>
<html>

<table>


Tables are very good for presenting data in an easily comparable, and viewable way. You can create a table in your HTML file by using the <table> tag. Within a table you need to put content. To place in content within the table, you will need 3 tags, <tr>, <th>, <td>. <th> stand for Table Heading, and as you can tell from the name, it creates a new table heading for a column. In your html file all the <th> elements are listed one after each other. <tr> stands for Table Row, it adds a new row in the table. Lastly, <td> stands for Table Data. The <td> element makes sure all the cells are in different columns but in the same row. However, in some contexts or situations you may need to create a table where a cell occupies 2 or more rows, or 2 or more columns; this is where the attributes (of the <td>) rowspan and colspancome in. By adding the attribute rowspan="2" or colspan="2"a cell will either occupy its own cell and the cell below, or its own cell and the cell to its right, respectively.
<html>
  <head>...</head>
  <body>
    <table>
      <tr> <!--This is a table row-->
        <th>I am a table heading</th>
      </tr>
      <tr>
        <td>I am table data cell</td>
      </tr>
  </body>
<html>

<br>


This tag essentially creates a linebreak in your page. Being and inline element, it must be placed within other elements, e.g. <p>, <h1>, etc..

<html>
  <head>...</head>
  <body>
    <p>
       Hello! This is my <br> linebroken paragraph.</p>
  </body>
<html>

<div>


This is probably the most important tag that you will EVER learn about. This tag essential creates a section of your webpage and marks it as your own. It is most often used to enclose many other similar elements that are in a section of your webpage, or parts that are in the same area. The use of a <div> will become apparent when learning CSS.

<html>
  <head>...</head>
  <body>
    <div>
      <h1>I am a primary heading enclosed by a div</h1>
      <p>I am a paragraph enclosed by a div</p>
    </div>
  </body>
<html>

Multi-page Websites

Linking


It is important to create a multi-page website so that your viewers can easily get to the information they want to read instead of having to scroll through the entire page! To create a multi-page website you must create links from which the user can go to the next page. The tag that you use to create a link is the <a> tag. Within the it, you need to specify an attribute: 'href'. This attribute is basically the route directions for the browser which tells it where to find the file that is being linked. When using the href attribute we use relative linking. Your link should look like this in your code if your are linking to a webpage about choc-chip cookies.
<html>
  <head>...</head>
  <body>
    <p>
      We all love <a href="cookie.html">choc-chip cookies!</a>
    </p>
  </body>
<html>

File organisation

Keeping eveything neat


As you start building larger and larger webpages, it is quite likely that the organisation and tidiness of your web domain, or web page 'folder' may have become minimal. However, it is important to organise files, media, and resources so that you can easily tell your HTML where to find the desired item, easily and logically. If you have organised your files, when telling your HTML where to find the image or video or file you use relative linking. Relative linking tells your browser whether to go in or out of files to find something.  A ‘/’ tells it to go in a folder, and a ‘../’ tells it to go out of a folder. Say, for example, we are on a webpage about cars, however, we want to step out of the car folder and look at some planes... we can!
<html>
  <head>...</head>
  <body>
    <p>
      <a href="Porsche/918.html">Porsche 918</a>
      <a href="../Planes/Boeing/787.html">Boeing 787 Dreamliner</a>
    </p>
  </body>
<html>

'Validation'

Ensuring reliability across (almost) all platforms


It is important to make sure your website will work on all browsers. A quick and easy way to check is to use a HTML validator. Essentially, this service scans through the code to ensure that no errors were committed in your code. You can make sure your website is XHTML 1.0 Strict or HTML5 by going to http://validator.w3.org and pasting the URL of your webpage and clicking check. Upon applying the suggested changes, revalidate your website, and voila! Your website should be offically support across all browsers.
<html>
  <head>...</head>
  <body>
    <p>
      <a href="Porsche/918.html">Porsche 918</a>
      <a href="../Planes/Boeing/787.html">Boeing 787 Dreamliner</a>
    </p>
  </body>
<html>
